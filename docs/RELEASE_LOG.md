# Release Log

历史发布流水已归档到 `docs/archive/2026-05-08-runtime-docs/RELEASE_LOG.md`。本文件只保留当前版本窗口和最近高信号变化。

## Unreleased

- v0.7 `归航航线` 命名差距读回：现有 `航线摘要`、详情差距和节奏预判会直接显示 `稳航校准` / `深空归航`，避免重复归航目标停留在抽象“下一段”。
- v0.7 `归航航线` 量化收益摘要：现有 `航线摘要` 和当前收益会把额外共鸣换算成“下轮起步可重建 N 台自动采集器”，让重复归航后的收益不只停留在抽象价值短语。
- v0.7 `归航航线` 价值摘要：现有 `航线摘要` 会在当前段、差距和节奏预判之外加入当前段价值短语，让重复归航不只读成门槛清单。
- v0.7 `归航航线` 航线摘要读回：现有 `共鸣矩阵` 会在完整细节前显示一行摘要，把当前段、距下一段差距和节奏预判压缩为首行扫读。
- v0.7 `归航航线` 航线图读回：现有 `共鸣矩阵` 会显示 `余辉起航 -> 稳航校准 -> 深空归航` 的完整三段路径和当前所在段；三段贯通后转为长期储备读回。
- v0.7 `归航航线` 节奏预判：现有 `共鸣矩阵` 会说明按当前路线再归航几次是否即可进入下一段，或归航次数达标后还差几点额外共鸣，避免把归航次数和共鸣门槛读成两条互不相关的任务。
- v0.7 `归航余辉` 已投入读回：两个永久节点已满且存在额外共鸣时，现有 `共鸣矩阵` 会持续显示余辉起步星尘和可支撑的自动采集器数量；即使起步星尘已被花掉，也会说明余辉已投入本轮重建节奏。
- v0.7 `归航航线` 当前收益读回：现有 `共鸣矩阵` 在下一段预览之外显示当前航线段已经带来的长期收益，区分起步星尘、重建时间压缩和后续版本储备。
- v0.7 `归航航线` 下一段预览：现有 `共鸣矩阵` 在行动提示之外显示达成当前门槛后会进入 `稳航校准`、`深空归航`，或在三段贯通后继续储备后续版本。
- v0.7 `归航航线` 行动提示：现有 `共鸣矩阵` 在差距读回之外显示下一步行动，区分继续归航、只补归航次数、只补额外共鸣和贯通后长期储备；阶段目标优先读回这段行动提示。
- v0.7 `归航航线` 动态差距读回：未贯通航线时，现有 `共鸣矩阵` 和阶段目标会显示距下一段还差几次归航、几点额外共鸣；三段航线贯通后转为后续版本长期储备读回。
- v0.7 `归航航线` 第一段可玩读回：已有归航历史且两个永久节点已启动后，现有 `共鸣矩阵` 显示当前航线、段落说明和下一段门槛；阶段目标优先读回当前航线。航线最多 3 段，纯派生自现有 `returnCount`、额外共鸣和永久节点。
- v0.7 `归航航线` 规划：新增 20 小时后长期目标预算，限定为现有 `共鸣矩阵` 和阶段目标里的最多 3 个航线里程碑；`governor-check` 现在会阻止缺失 v0.7 预算的扩展。
- `归航余辉` 节奏读回：现有 `共鸣矩阵` 现在会把余辉起步星尘换算成可立即重建的自动采集器数量，并用结构化读回块显示“起步星尘”和“可重建”；50 星尘上限对应前 3 台自动采集器。
- v0.6 `归航余辉`：两个永久节点已满后，继续归航获得的额外共鸣会让新一轮以少量星尘起步，最高 50 星尘，并在现有 `共鸣矩阵` 和阶段目标中读回。
- 两个永久节点已满后，如果继续归航并带着额外共鸣回到新一轮，现有 `共鸣矩阵` 会说明额外共鸣暂存到后续版本，阶段目标也提示“额外共鸣已暂存”。
- 归航后第 2 轮开局现在会在已有可用共鸣、永久节点或归航历史时继续显示现有 `共鸣矩阵`，并优先提示用共鸣启动永久节点。
- 新增 runtime 文档预算政策并压缩当前运行态文档；历史细节归档到 `docs/archive/2026-05-08-runtime-docs/`，保留 Issue #1/#2 decision anchors 和当前 v0.5 归航方向。
- `governor-check` 对运行态文档同时检查行数和文件大小，避免超长单行继续制造文档膨胀。
- 新增 harness engineering scorecard：用 `docs/QUALITY_SCORE.md` 记录 agent readability、content depth、mechanical checks 和 garbage collection，并由 `governor-check` 校验结构。

## Current Playable State

- v0.5 `星尘归航` 第一版已实现：达到 25 台自动采集器和 15 次调校后，现有 `共鸣矩阵` 显示 `星尘归航 +1 共鸣`。
- 归航重启本轮工坊，奖励固定 `1 共鸣`，保留共鸣、已启动永久节点、已领取共鸣里程碑和归航次数。
- 归航后有可用共鸣时，新一轮开局继续显示现有 `共鸣矩阵`，阶段目标优先提示用共鸣启动永久节点；两个永久节点已满后，额外共鸣会点亮 capped `归航余辉`，并读回可重建的自动采集器数量。
- v0.7 `归航航线` 第一段、具名动态差距读回、行动提示、具名节奏预判、航线图、带量化收益的航线摘要、下一段预览、当前收益读回和余辉已投入读回已可玩；下一步应复核航线是否真正减少重复归航后的迷失感，仍不新增第三共鸣门槛、新节点、节点等级树、任务系统、多生产线或额外面板。
- `共鸣矩阵` 仍保持最多 2 个永久节点，不新增第三共鸣门槛、新节点、节点等级树、任务系统、多生产线或额外面板。
