# v0.7 归航航线行动提示复核设计

## 背景

上一轮 `归航航线` 已显示当前段、下一段门槛和动态差距。复核后仍有一个可读性缺口：玩家能看到还差多少，但在只缺归航次数、只缺额外共鸣、或两者都缺时，还要自己推断下一步应该继续归航、保留共鸣，还是单纯等待后续版本。

## 方案

继续把 v0.7 限制为纯派生读回：

- `getReturnRouteReadback` 增加 `actionHint`，只从 `returnCount`、额外共鸣和已启动永久节点推导。
- 未贯通航线时，行动提示区分三种状态：归航次数和额外共鸣都不足、只缺归航次数、只缺额外共鸣。
- 航线贯通后，行动提示转为继续归航并储备额外共鸣。
- `共鸣矩阵` 显示行动提示；阶段目标优先使用行动提示，而不是只复述差距。

## 取舍

选择行动提示，而不是降低门槛或新增奖励。这样能把“差多少”变成“下一步做什么”，但不改变数值节奏、奖励公式、存档字段或 UI 面板数量。

## 验证

- `src/return-route.test.ts` 覆盖两者都缺、只缺归航次数、只缺额外共鸣和已贯通航线的行动提示。
- `src/App.test.tsx` 覆盖 `共鸣矩阵` 和阶段目标的行动提示。
- 标准链路：`bun test`、`bun run test`、`bun run build`、`./ops/governor-check.sh`、`git diff --check`。
