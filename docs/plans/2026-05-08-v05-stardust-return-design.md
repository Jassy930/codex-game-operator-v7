# v0.5 星尘归航设计

## 背景

当前 v0.4 已经把内容推到第二共鸣和约 20 小时目标，但它更像长线目标骨架：玩家继续购买自动采集器和调校工具，达到门槛后领取共鸣、启动第二个永久节点，然后又回到等待下一次普通升级。

用户明确指出：当前内容支撑不了 20 小时游玩，并且对 prestige 作为游戏长线主机制更感兴趣。这个判断合理。Idle / incremental 游戏的长线通常需要一个 reset / prestige 循环，把前一轮积累转化成下一轮更强的永久成长。

因此 v0.5 的主机制从 `星图巡航` 调整为 `星尘归航`：一个轻量 prestige 循环。`共鸣` 不再只是里程碑奖励，而升级为归航后的长期奖励资源。

## 目标

让 20 小时目标之后形成真正的长线循环：

- 玩家推进星尘工坊。
- 玩家达到归航条件。
- 玩家选择归航，重启本轮工坊。
- 玩家获得 `共鸣`。
- 玩家用 `共鸣` 解锁或强化永久节点。
- 下一轮更快、更清楚地推进到更深阶段。

## 非目标

- 不新增第三普通资源。
- 不新增复杂地图、多生产线、角色队伍或装备系统。
- 不做任务列表、每日任务、成就清单或委托系统。
- 不做付费、广告、限时活动或外部 telemetry。
- 不让前 60 秒变复杂。
- 第一版不做完整多层 prestige 树。

## 核心命名

### 星尘归航

`星尘归航` 是本项目里的 prestige 名称。它表达的是：工坊把本轮星尘结构折返成更稳定的长期共鸣。

UI 文案不使用 `prestige`。玩家看到的是：

```text
星尘归航：重启本轮工坊，获得 1 共鸣
```

### 共鸣

`共鸣` 作为 prestige 奖励资源继续保留。

设计决策：

- 不新增 `星核`、`星魂`、`归航点` 等第三资源。
- 归航奖励直接给 `共鸣`。
- 现有 `共鸣矩阵` 成为归航后的永久成长面板。
- 后续可以扩展共鸣节点或节点等级，但第一版不做复杂树。

## 核心循环

```text
采集星尘
→ 购买自动采集器 / 调校工具
→ 进入星尘引擎室
→ 达成归航条件
→ 星尘归航，获得共鸣
→ 花共鸣启动永久节点
→ 新一轮更快推进
```

这个循环比继续增加第二、第三共鸣门槛更适合支撑长线，因为它让“重开一轮”本身变成成长，而不是单纯等待更大数字。

## 第一版归航条件

v0.5 第一版只定义一个归航条件：

- 已达到 `星尘引擎室`。
- 自动采集器不少于 25。
- 调校等级不少于 15。
- 当前已经领取过首个共鸣，理解了共鸣矩阵。

这复用 v0.4 的第二共鸣门槛，但语义改变：

- v0.4：到达门槛后领取第二共鸣。
- v0.5：到达门槛后可以执行一次 `星尘归航`。

第一版不再新增第三共鸣门槛。

## 节奏目标

下面的时间来自当前经济参数的主动 self-playtest 模型：每秒点击一次，并在可购买时优先购买当前更便宜的自动采集器或调校工具。它不是承诺给玩家看的精确时间，而是设计校准值。

| 状态 | 预计触达 | 设计含义 |
| --- | --- | --- |
| 首个共鸣：20 自动采集器 / 12 调校 | 约 3.5-3.8 小时 | 玩家第一次接触长期资源和共鸣矩阵 |
| 首次归航：25 自动采集器 / 15 调校 | 约 16-18 小时 | 第一次 prestige 入口，支撑 20 小时目标 |
| 已有 `稳定回路` 后再次到达归航门槛 | 约 16.6 小时 | 第一轮永久节点应该让下一轮略快，但不直接压缩成短局 |
| `稳定回路 + 调校刻印` 后再次到达归航门槛 | 约 16.3 小时 | 双节点应有可感知但克制的提速 |

设计结论：

- v0.5 第一版不追求让下一轮大幅缩短，而是让 20 小时后出现可重复的长期循环。
- 如果归航后第二轮仍显得空，优先加“归航后目标读回 / 节点等级 / 归航事件”，而不是直接把奖励倍率拉高。
- 如果真实玩家无法触达首个共鸣，问题在 3-4 小时节奏，不应通过降低归航门槛解决。

## 归航奖励

第一版奖励：

```text
获得 1 共鸣
```

奖励原则：

- 共鸣是长期资源，归航后保留。
- 已启动共鸣节点保留。
- 归航次数保留。
- 星尘、自动采集器、调校等级回到新一轮初始状态。

后续可考虑基于超额进度提高奖励，例如 30 台自动采集器或 18 次调校给更多共鸣，但第一版不做，避免玩家为了最优收益被迫等待。

## 重置范围

归航会重置本轮工坊：

- `dust`
- `autoCollectors`
- `dustPerSecond`
- `nextAutoCollectorCost`
- `autoCollectorEfficiencyLevel`
- `autoCollectorEfficiencyMultiplier`
- `nextEfficiencyUpgradeCost`
- `lastUpdatedAt`

归航会保留长期状态：

- `resonance`
- `earnedResonanceMilestones`
- `unlockedResonanceNodes`
- `returnCount`
- metrics / feedback local-only 状态

第一版不保留 `activeCruiseRoute`，因为星图巡航不再是 v0.5 主机制。

关键规则：

- `earnedResonanceMilestones` 不随归航重置。
- `first-resonance` 是一次性的共鸣教学入口。
- v0.5 后，重复获得共鸣主要来自 `星尘归航`，不是重复领取首个共鸣。

## 共鸣矩阵调整

现有 `共鸣矩阵` 不废弃，它变成长线成长面板。

第一版矩阵规则：

- 归航获得共鸣。
- 玩家可用共鸣启动永久节点。
- 已启动节点跨归航保留。
- 现有节点效果继续成立：
  - `稳定回路`：自动采集产出提高。
  - `回访线圈`：离线收益提高。
  - `调校刻印`：调校倍率提高。

第一版不新增节点等级。节点等级是后续候选，不放进第一刀。

## 现有第二共鸣如何处理

v0.5 应避免同时存在“第二共鸣里程碑”和“归航奖励共鸣”两个相似入口。

推荐迁移：

- 首个共鸣仍可作为玩家第一次接触长期资源的里程碑。
- 第二共鸣门槛改为 `归航条件`。
- 达成 25 自动采集器 / 15 调校后，按钮不再是“领取第 2 点共鸣”，而是“星尘归航 +1 共鸣”。
- 如果玩家已有旧存档并已领取 `second-resonance`，可以视为已经获得那一点共鸣，不强制迁移回滚。
- v0.5 实现时应停止生成新的 `second-resonance` 领取入口；旧存档中的字符串只作为历史兼容存在。

这样能保留当前玩家进度，又把新设计方向统一到 prestige。

## 轮次结构

### 第 0 轮：建立共鸣概念

玩家从新存档开始：

1. 通过星尘、自动采集器和调校工具进入 `星尘引擎室`。
2. 达到 `20 自动采集器 / 12 调校`。
3. 领取首个共鸣。
4. 在 `共鸣矩阵` 启动第一个永久节点。

这一轮的重点是理解 `共鸣` 和永久节点，不急着解释 prestige。

### 第 1 轮：首次归航

玩家继续推进到 `25 自动采集器 / 15 调校`：

1. `共鸣矩阵` 显示归航准备进度。
2. 条件满足后显示 `星尘归航 +1 共鸣`。
3. 归航后本轮工坊重启，保留共鸣、永久节点和归航次数。
4. 玩家可以用归航获得的共鸣启动第二个永久节点。

这一轮的重点是让玩家明白：重启不是惩罚，而是把本轮结构压缩成长期共鸣。

### 第 2 轮及以后：重复长线

后续每轮继续以 `25 / 15` 作为归航门槛，奖励 1 共鸣。

第一版不会立即提供更多节点或节点等级，但会在文档里预留后续方向：

- 共鸣节点等级。
- 归航后事件反馈。
- 星图巡航作为本轮策略倾向。
- 更高阶归航条件和奖励公式。

## UI 设计

### 未达成条件

在 `共鸣矩阵` 或现有阶段目标里显示：

```text
归航准备：自动采集器 23/25，调校 14/15
```

### 可归航

显示一个明确按钮：

```text
星尘归航 +1 共鸣
```

按钮说明：

```text
重启本轮工坊，保留共鸣和永久节点，让下一轮更快进入引擎室。
```

### 归航后

事件反馈：

```text
星尘归航完成：获得 1 共鸣，工坊回到新一轮火花工作台
```

阶段目标：

```text
归航目标：用共鸣启动永久节点，再推进下一轮工坊
```

## 数值原则

第一版归航必须轻，但要有实感：

- 归航奖励固定为 1 共鸣。
- 不因为多等几个小时给更多共鸣。
- 不改变自动采集器和调校工具成本曲线。
- 通过永久节点让下一轮自然更快。
- 归航奖励不读取当前星尘余额，避免玩家为了最优收益卡在归航前。
- 归航门槛只看结构进度：自动采集器数量和调校等级。

如果 self-playtest 显示下一轮仍太慢，优先微调现有永久节点效果或增加节点等级，而不是提高归航奖励。

## 状态与文案

| 状态 | 条件 | 玩家看到 |
| --- | --- | --- |
| 尚未接触共鸣 | 未达到 20/12 | 不显示归航 |
| 首个共鸣可领取 | 达到 20/12，未领取 `first-resonance` | `共鸣目标：领取首个共鸣，再选择 1 个永久节点` |
| 归航准备中 | 已领取首个共鸣，未达到 25/15 | `归航准备：自动采集器 X/25，调校 Y/15` |
| 可归航 | 已领取首个共鸣，达到 25/15 | `星尘归航 +1 共鸣` |
| 归航后有可用共鸣 | 归航完成，`resonance > 0` 且还有可启动节点 | `归航目标：用共鸣启动永久节点，再推进下一轮工坊` |
| 节点已满 | 已启动 2 个节点 | `归航目标：永久节点已形成，继续归航积累共鸣，等待后续节点等级` |

第一版允许“归航后共鸣暂时没有可消费点”的状态存在，但必须读回清楚，不能让玩家以为出错。

## 存档设计

v0.5 需要 save version 3。

新增字段：

```ts
returnCount: number
```

可选后续字段，第一版不加：

```ts
lastReturnAt: number
bestReturnTimeMs: number
```

兼容策略：

- v1 / v2 存档加载时默认 `returnCount = 0`。
- 已有 `resonance`、`earnedResonanceMilestones`、`unlockedResonanceNodes` 按当前逻辑保留。
- 旧存档里已领取 `second-resonance` 不回滚。
- `second-resonance` 不再作为新版本的可领取目标生成。
- 如果旧存档已领取 `second-resonance` 且拥有额外共鸣，保留该共鸣。

## 指标边界

第一版可以新增 local-only 指标，但不强制。

候选指标：

- `returnCount`
- `firstReturnTimeMs`

不上传、不跨设备、不记录个人数据。若实现指标，仍通过现有 `src/metrics.ts` 和 localStorage。

## 星图巡航的位置

`星图巡航` 不删除，但降级为后续辅助系统：

- 它可以成为归航后的本轮策略选择。
- 它不再是 v0.5 的主长线机制。
- 只有当归航循环稳定后，才考虑把星图巡航作为“本轮航线倾向”加入。

也就是说：

```text
v0.5 主线：星尘归航
后续辅助：星图巡航
```

## 方案比较

### 方案 A：共鸣作为归航奖励资源

优点：

- 复用玩家已见过的第二资源。
- 共鸣矩阵自然变成长线成长面板。
- 不新增第三资源。
- 能形成标准 idle / incremental 长线循环。

风险：

- 需要重新定义第二共鸣门槛的语义。
- 归航重置必须讲清楚，否则玩家会担心丢进度。

### 方案 B：新增归航资源

优点：

- prestige 资源语义更独立。

风险：

- 星尘、共鸣、归航资源三层过快变复杂。
- 现有共鸣矩阵会被边缘化。
- 前期解释成本过高。

### 方案 C：保持星图巡航为主机制

优点：

- 不需要 reset。
- 风险较低。

风险：

- 仍然缺少真正的长线循环。
- 难以支撑 20 小时以上的可重复玩法。

## 结论

采用方案 A：`共鸣` 作为 `星尘归航` 的奖励资源。

v0.5 第一版只做：

- v0.5 prestige 预算。
- `returnCount` 存档字段。
- 归航条件。
- 归航按钮和确认反馈。
- 归航后重置本轮工坊并获得 1 共鸣。
- 保留共鸣和永久节点。

不做新资源、节点等级、复杂树、星图巡航、任务系统、多生产线或 prestige 奖励公式。
