# v0.8 归航航线贯通奖励设计

## 背景

v0.7 已经把 `归航航线` 做成最多 3 段的长期目标，并让航线里程碑轻量降低后续归航准备门槛。当前缺口是三段贯通后仍偏读回层，玩家看到 `深空信标储备`，但完成态缺少一个能直接影响长线循环的收益。

## 方案

三段 `归航航线` 贯通后，后续 `星尘归航` 奖励从 1 共鸣提升到 2 共鸣。

- 三段贯通前：`星尘归航` 仍奖励 1 共鸣。
- 三段贯通后：`星尘归航` 奖励 2 共鸣。
- 奖励只从现有 `returnCount`、额外共鸣和已启动永久节点派生。
- 归航台按钮、归航完成事件和 `归航航线` 当前收益同步读回奖励变化。

## 边界

- 不新增第三资源、节点、节点等级树、任务系统或面板。
- 不新增 `GameState` 字段，不修改 localStorage 存档格式。
- 不改变 `归航余辉` 公式、50 星尘上限或归航准备门槛。
- 不新增 telemetry、视觉资产或依赖。

## 验证

- `src/return.test.ts` 覆盖贯通前 +1、贯通后 +2 和归航后共鸣增长。
- `src/return-route.test.ts` 覆盖完成态当前收益读回 +2。
- `src/App.test.tsx` 覆盖归航台按钮和归航完成事件显示 +2。
- 标准收口仍运行 `bun test`、`bun run test`、`bun run build`、`./ops/governor-check.sh` 和 `git diff --check`。
