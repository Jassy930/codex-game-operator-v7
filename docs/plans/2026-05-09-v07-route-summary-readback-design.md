# v0.7 归航航线摘要读回设计

## 背景

`归航航线` 已经补齐当前段、收益、下一段门槛、差距、行动提示、节奏预判、航线图和下一段预览。信息完整，但矩阵内读回行数较多，玩家需要扫完整块才能拼出“我现在在哪、差什么、按当前节奏能不能推进”。

## 本轮选择

在现有 `ReturnRouteReadback` 中新增纯派生 `routeSummary`：

- 未贯通时，摘要压缩为当前航线段、动态差距和节奏预判。
- 已贯通时，摘要压缩为三段贯通和长期储备。
- UI 只在现有 `共鸣矩阵` 的 `归航航线` 块内把摘要放在段落说明之前。

## 明确不做

- 不新增资源、节点、节点等级、任务系统、多生产线、额外面板、存档字段、视觉资产或 telemetry。
- 不修改 `星尘归航` 奖励。
- 不修改 `归航余辉` 公式或 50 星尘上限。
- 不改变 v0.7 最多 3 段航线的复杂度边界。

## 验证

- `src/return-route.test.ts` 覆盖摘要字段。
- `src/App.test.tsx` 覆盖矩阵 SSR 读回。
- 完整运行 `bun test`、`bun run test`、`bun run build`、`./ops/governor-check.sh`、`git diff --check`。
