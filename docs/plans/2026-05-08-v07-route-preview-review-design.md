# v0.7 归航航线下一段预览复核设计

## 背景

上一轮 `归航航线` 已把差距读回转成行动提示，玩家能知道下一步应继续归航、保留额外共鸣，还是补足单一缺口。复核后仍有一个可读性缺口：玩家能看到“下一段：累计 N 次归航，并保留 N 点额外共鸣”，但还要自己推断达成后会进入哪一段航线，以及那一段代表什么长期方向。

## 方案

继续把 v0.7 限制为纯派生读回：

- `getReturnRouteReadback` 增加 `nextPreview`，只从当前航线阶段派生。
- 第一段显示达成后进入 `稳航校准`，强调余辉重建节奏会被稳定成长期航标。
- 第二段显示达成后进入 `深空归航`，强调后续归航会转为长期储备。
- 三段贯通后显示没有下一段，继续归航储备后续版本。
- `共鸣矩阵` 显示下一段预览；阶段目标仍保留行动提示，避免目标行过长。

## 取舍

选择下一段预览，而不是新增奖励或降低门槛。这样能让 20 小时后的重复归航更像一条可理解的路线，但不改变数值节奏、奖励公式、存档字段或 UI 面板数量。

## 验证

- `src/return-route.test.ts` 覆盖第一段、第二段和贯通后的下一段预览。
- `src/App.test.tsx` 覆盖 `共鸣矩阵` 显示下一段预览。
- 标准链路：`bun test`、`bun run test`、`bun run build`、`./ops/governor-check.sh`、`git diff --check`。
