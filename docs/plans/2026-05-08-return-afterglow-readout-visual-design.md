# 归航余辉读回视觉设计

## 背景

当前 v0.6 `归航余辉` 已把额外共鸣换算成 capped 起步星尘，并进一步读回“可立即重建几台自动采集器”。现有 UI 仍把这段价值放在普通提示行里，视觉层级接近说明文字，玩家不容易把它当成一次归航后的关键收益。

## 目标

在不改数值、不改存档、不新增图片和不新增面板的前提下，让 `归航余辉` 成为 `共鸣矩阵` 内的稳定读回块：

- 读回起步星尘。
- 读回可立即重建的自动采集器台数。
- 保留现有完整说明文案，避免测试和玩家理解断层。
- 保持移动端和桌面端紧凑，不遮挡归航按钮、节点选择或反馈入口。

## 方案

复用现有 `共鸣矩阵`：

- 将 `hasActiveReturnAfterglow` 分支从普通段落升级为 `.return-afterglow-readout`。
- 读回块内部使用两个紧凑数据格：`起步星尘` 和 `可重建`。
- 使用现有色彩体系，偏暖色强调“归航收益”，不引入新图片或图标。
- 保留原句“共鸣余辉：额外共鸣让新一轮从 X 星尘起步，可立即重建 N 台自动采集器”作为说明。

## 不做

- 不生成新图片。
- 不新增普通资源、共鸣节点、节点等级、任务系统、多生产线或额外面板。
- 不修改余辉公式、50 星尘上限、存档字段或 telemetry。

## 验证

- SSR 测试确认读回块 class 和读回标签存在。
- CSS 测试或构建确认样式可解析。
- 运行 `bun test`、`bun run build`、`./ops/governor-check.sh`、`git diff --check`。
