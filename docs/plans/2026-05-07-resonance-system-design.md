# v0.3 共鸣系统设计

## 背景

v0.2 已经完成 3-15 分钟内容弧线、工坊阶段、调校工具、首次回访和回访后消费目标。用户明确指出：下一阶段不应只增加里程碑，而应有更丰富的玩法。

上一轮 `DECISION:2026-05-07-research-milestone-resonance` 把 `星尘共鸣节点` 作为候选方向，但仍停留在预告层。v0.3 应把“里程碑”升级为一个真实系统：里程碑是入口，共鸣是资源，矩阵是选择空间。

## 目标

让玩家在首次回访后获得新的策略层：

- 不只继续攒星尘买自动采集器或调校工具。
- 能通过长期生产节点获得第二资源 `共鸣`。
- 能在 `共鸣矩阵` 中选择永久强化方向。
- 仍保持 idle / incremental 的轻量结构，不一次性引入 prestige、任务系统或多面板管理。

## 非目标

- 不做 prestige / 重置循环。
- 不做任务系统、每日任务或委托。
- 不做多地图、多角色或多生产线。
- 不做外部 analytics、上传 telemetry 或玩家身份。
- 不做付费、广告、限时活动或抽卡。

## 设计方案

### 核心循环

v0.3 的新增循环是：

1. 玩家继续采集星尘。
2. 玩家购买自动采集器和调校工具。
3. 达到共鸣门槛时获得 `共鸣`。
4. 玩家花费共鸣解锁矩阵节点。
5. 节点提供永久强化，反过来影响星尘生产和回访收益。

### 第二资源：共鸣

`共鸣` 是 v0.3 的第二资源。它不是普通货币，也不通过点击直接获得。

获得方式第一版只使用派生里程碑：

- 首个共鸣：达到 `20 台自动采集器 + 12 次调校`。
- 后续共鸣：暂不实现，先在设计中预留门槛序列。

这样可以确保第二资源从长期生产成就中出现，而不是抢走前 60 秒核心目标。

### 新面板：共鸣矩阵

`共鸣矩阵` 是一个新增轻量面板，只在玩家达到 `星尘引擎室` 后显示。

第一版节点：

- `稳定回路`：自动采集器基础产出 +10%。
- `回访线圈`：离线收益结算 +10%。
- `调校刻印`：调校倍率额外 +5%。

第一版只允许解锁 1 个节点，避免一次性扩复杂度。后续版本再允许更多节点和分支。

### 存档版本

v0.3 需要 save version 2。

新增字段：

- `resonance`: 当前共鸣数量。
- `earnedResonanceMilestones`: 已领取的共鸣里程碑 id。
- `unlockedResonanceNodes`: 已解锁的矩阵节点 id。

兼容策略：

- v1 存档升级到 v2 时保留星尘、自动采集器、调校等级和成本。
- v1 缺失字段时默认 `resonance = 0`、两个数组为空。
- 仍保留离线收益结算逻辑。

### 本地指标

指标仍只保存在 localStorage。

新增 local-only 字段：

- `resonanceEarnedCount`
- `resonanceNodeUnlockedCount`
- `firstResonanceTime`

不上传、不跨设备、不记录个人数据。

### UI

首屏继续以星尘、采集、自动采集器、调校工具为主。

共鸣矩阵显示规则：

- `星尘引擎室` 前不显示矩阵。
- 达到 `星尘引擎室` 后显示一个紧凑面板。
- 未达到首个共鸣门槛时显示当前进度：`自动采集器 X/20，调校 Y/12`。
- 获得共鸣后显示可解锁节点。

## 复杂度边界

v0.3 可以放开：

- 第二资源：最多 1 个。
- 新面板：最多 1 个。
- 存档版本：允许 v2。
- 本地指标字段：允许与共鸣相关的最多 3 个字段。

v0.3 仍禁止：

- prestige。
- 任务系统。
- 多地图或多生产线。
- 多个新面板。
- 外部 analytics 或上传 telemetry。

## 方案比较

### 方案 A：共鸣系统

优点：

- 形成真实第二资源和策略选择。
- 能解释为什么玩家要继续扩建和调校。
- 可从现有状态派生，前 60 秒不受影响。

风险：

- 需要存档迁移和更多测试。
- 新面板必须保持紧凑，避免首屏过载。

### 方案 B：直接 prestige

优点：

- 长期目标更强。
- 能形成重置后的成长曲线。

风险：

- 当前过早。游戏还没有足够中期系统支撑重置收益。
- 容易让玩家误解为“之前进度被清空”。

### 方案 C：任务系统

优点：

- 目标明确，内容扩展容易。

风险：

- 容易变成 checklist，而不是 idle / incremental 的经济选择。
- 会新增任务状态、任务 UI 和大量文案维护。

## 结论

采用方案 A：v0.3 共鸣系统。

第一版只实现一个最小闭环：

- v2 存档。
- 第二资源 `共鸣`。
- 首个共鸣里程碑。
- 一个紧凑 `共鸣矩阵` 面板。
- 三个节点中允许解锁一个。
- local-only 共鸣指标。

prestige 和任务系统继续停放，等待 v0.3 实测后再评估。
