# 移动端按钮视觉打磨设计

## 背景

当前页面已经有星尘背景、资源插图和升级插图。截图检查显示桌面端整体可读，但移动端首屏的操作按钮被纵向拉得过高，主要原因是按钮在窄屏纵向排列时仍继承 `flex: 1 1 180px` 的高度基准。

## 目标

优化现有按钮层级和移动端首屏节奏：

- 移动端按钮高度回到紧凑、稳定的操作尺寸。
- 保留星尘、自动采集器和调校工具插图，不新增图片。
- 保持现有文案、按钮数量、面板数量、资源数量和玩法逻辑不变。
- 让不可购买升级按钮的视觉状态更明确，但不降低文字可读性。

## 方案

只调整 CSS：

- 在 `max-width: 560px` 媒体查询中覆盖 `.primary-action` 和 `.upgrade-action` 的 `flex-basis`。
- 为移动端按钮设置稳定 `width`、`min-height` 和内边距。
- 在移动端将升级按钮内容左对齐，避免图标和长文案在窄屏里显得漂浮。
- 对 disabled 升级按钮里的图片做轻度降饱和和透明度处理，强化状态区分。

## 不做

- 不生成新图片。
- 不新增共鸣矩阵视觉节点。
- 不改变 React 状态、经济公式、存档、指标或反馈入口。
- 不新增面板、按钮或资源。

## 验证

- `src/ops-scripts.test.ts` 读取 CSS，确认移动端按钮覆盖了 `flex-basis: auto`。
- 截取移动和桌面页面，确认按钮不再占据异常高度。
- 运行 `bun test`、`bun run build`、`./ops/governor-check.sh`、`git diff --check`。
