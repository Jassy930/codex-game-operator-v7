# 工坊物件插图设计

## 背景

当前 v0.2 重点是 3-15 分钟内容弧线。游戏已经有单资源、自动采集器、调校工具和工坊阶段，但核心链路主要依赖文字和按钮表达。用户确认选择“资源/升级插图”方案，希望用图片优化现有画面表现。

## 设计目标

使用少量项目内位图素材强化核心链路：

- 星尘：对应主资源读数。
- 自动采集器：对应第一个自动化升级。
- 调校工具：对应效率调校升级。

素材只作为现有 UI 的视觉锚点，不新增资源、按钮、面板、任务系统、地图、角色或 lore。

## 方案

使用 `imagegen` 生成 3 张一致风格的小型插图，保存到 `src/assets/` 并由现有 React UI 引用。

- 插图尺寸目标为正方形，使用稳定 CSS 尺寸约束。
- 风格为温暖、清晰、轻量科幻工坊，贴合现有星尘工坊主题。
- 图片不得包含文字、水印、真实品牌、真实人物或复杂背景。
- 资源区加入星尘插图，两个升级按钮加入对应插图。
- 保持现有按钮文案、禁用状态、进度、目标、里程碑和反馈入口不变。

## 约束

- 不增加 visible panel。
- 不增加 upgrade type。
- 不增加 primary/secondary resource。
- 不新增指标字段。
- 不使用远程热链。
- 图片资产必须保存在项目内，并更新相关文档和决策记录。

## 验证

- 组件测试确认三类图片语义出现在首屏 HTML 中。
- CSS/构建确认资产被项目引用。
- 运行 `bun test`、`bun run build`、`./ops/governor-check.sh`。
