# v0.5 星图巡航设计

## 状态

本方案已降级为后续辅助内容，不再作为 v0.5 主线机制。

用户明确希望 prestige 成为游戏长线主机制，并倾向把现有第二资源 `共鸣` 作为 prestige 奖励资源。因此 v0.5 主设计改为：

- [v0.5 星尘归航设计](./2026-05-08-v05-stardust-return-design.md)
- [v0.5 Stardust Return Implementation Plan](./2026-05-08-v05-stardust-return.md)

`星图巡航` 后续可作为归航后的本轮策略辅助，但不应抢在 `星尘归航` 之前实现。

## 原背景

v0.4 把首个共鸣后的长期目标延长到约 20 小时：第二共鸣门槛、第二个现有永久节点、双节点组合读回和当前版本完成读回都已经落地。

但这仍然主要是“更远门槛 + 更清楚文案”。它能解释 20 小时目标，却不足以支撑 20 小时的实际游玩内容。玩家在第二共鸣后仍主要等待自动采集器和调校工具的下一次购买，没有新的决策节奏。

v0.5 的目标是引入一个真正的新内容层，但仍保持轻量 idle / incremental 结构。

## 目标

让第二共鸣后的玩家获得新的长期玩法：

- 不只是看见“当前版本目标已完成”。
- 能基于已启动的双节点组合选择一个巡航方向。
- 巡航方向改变后续 20 小时内的目标、收益倾向和回访计划。
- 第一版只做一个可理解、可测试、可继续扩展的最小闭环。

## 非目标

- 不做 prestige / 重置循环。
- 不做任务系统、每日任务、成就清单或委托列表。
- 不做多地图、多生产线、多角色或复杂探索地图。
- 不新增第三共鸣门槛或新共鸣节点来冒充新内容。
- 不新增外部 analytics、上传 telemetry 或个人数据。
- 不改变前 60 秒核心循环。

## 核心概念

### 星图巡航

`星图巡航` 是第二共鸣后解锁的长期路线选择。

它不是地图系统。第一版只是一条嵌入现有 `共鸣矩阵` 的长期路线状态：

1. 玩家领取第二共鸣。
2. 玩家启动第 2 个永久节点。
3. `共鸣矩阵` 出现 `星图巡航` 选择。
4. 玩家选择一个巡航方向。
5. 该方向改变长期目标文案，并给一个小幅、可解释的长期倾向效果。

### 巡航方向

第一版只根据现有双节点组合提供 2 到 3 个方向，不新增资源。

可用方向：

- `采集航线`：偏向自动采集器，提升长期生产目标感。
- `回访航线`：偏向离线收益和回访节奏。
- `调校航线`：偏向调校工具和倍率成长。

方向不是任务。它是玩家的长期策略姿态。

### 路线组合

巡航路线来自玩家已经做出的双节点选择。这样路线不是凭空出现，而是对前面 20 小时策略的延伸。

| 已启动节点组合 | 可选巡航路线 | 设计意图 |
| --- | --- | --- |
| 稳定回路 + 回访线圈 | 采集航线 / 回访航线 | 让玩家在在线增长和离线回访之间选长期倾向 |
| 稳定回路 + 调校刻印 | 采集航线 / 调校航线 | 让玩家在继续堆采集器和继续提高效率之间选长期倾向 |
| 回访线圈 + 调校刻印 | 回访航线 / 调校航线 | 让玩家在低频回访和高倍率调校之间选长期倾向 |

第一版不提供与未启动节点无关的路线。例如玩家没有启动 `回访线圈`，就不显示 `回访航线`。这能让路线选择读起来像“你的工坊已经形成这种能力”，而不是通用菜单。

## 玩家旅程

### 0-60 秒

完全不出现星图巡航。玩家仍只看到星尘、采集、自动采集器、调校工具和早期目标。

### 3-15 分钟

仍然是工坊阶段和下一升级目标。星图巡航不提前预告，避免让早期玩家看到太多远期名词。

### 首次回访后

共鸣矩阵出现，玩家先理解 `共鸣` 和第一个永久节点。此时仍不显示星图巡航。

### 第二共鸣后

玩家启动第 2 个永久节点，看到 2/2 选择上限和当前版本共鸣目标完成。此时星图巡航成为新的内容入口：

```text
星图巡航：选择一条长期航线，让双节点组合进入下一段运行节奏
```

### 选择巡航后

玩家看到路线被固定，并且主目标行开始用路线解释后续等待：

```text
星图巡航：回访航线正在放大离线收益，离开一会儿后回来投入下一次升级
```

这一步的关键不是给玩家一个复杂任务列表，而是给 20 小时之后的工坊一个新的长期姿态。

## 第一版玩法

### 解锁条件

同时满足：

- 已领取首个共鸣。
- 已领取第二共鸣。
- 已启动 2 个现有永久节点。
- 当前没有已选择的巡航方向。

### 选择规则

- 第一版只允许选择 1 条巡航方向。
- 选择后写入存档。
- 选择后可读回，但不能频繁切换。
- 不新增按钮行；复用 `共鸣矩阵` 的紧凑区域。
- 第一版不提供重置路线或切换路线，避免玩家把它当成短期开关。

### 第一版效果

效果必须轻，不破坏经济：

- `采集航线`：自动采集器产出额外 +5%。
- `回访航线`：离线收益额外 +5%。
- `调校航线`：有效调校倍率额外 +0.03。

这些效果比共鸣节点弱，作用是让路线选择被感知，而不是开新系统。

### 航段目标

为避免星图巡航只变成“第三种永久 buff”，v0.5 设计为可继续扩成航段目标。

第一版只实现 `activeCruiseRoute`，但文案要为后续航段留下空间：

- `采集航线` 的后续航段可以要求继续扩建自动采集器。
- `回访航线` 的后续航段可以要求一次有效离线回访。
- `调校航线` 的后续航段可以要求继续调校到新的效率区间。

第一版不新增 `cruiseStage` 字段。等路线选择和效果被验证后，再考虑 v0.5 第二刀加入航段进度。这样可以避免一次性引入路线、进度、事件和奖励四个概念。

### 目标读回

选择后，`星尘引擎室` 阶段目标应显示：

- 当前巡航方向。
- 它正在强化哪类长期收益。
- 下一次更近升级或回访建议。

示例：

```text
星图巡航：采集航线正在放大自动采集，约 42 分钟后可购买自动采集器
```

路线读回优先级：

1. 如果有可领取共鸣，仍优先提示领取。
2. 如果有可用共鸣且还能启动第 2 个节点，仍优先提示选择节点。
3. 如果已经选择巡航路线，显示星图巡航目标。
4. 否则显示现有回访计划、20 小时巡航或完成状态。

## UI 设计

星图巡航嵌入现有 `共鸣矩阵`，不新增页面级面板。

未选择路线时：

```text
星图巡航
选择一条长期航线，让双节点组合进入下一段运行节奏
[采集航线] 自动采集长期产出 +5%
[回访航线] 离线收益 +5%
```

已选择路线时：

```text
星图巡航：回访航线已设定
离线收益 +5% · 回访线圈会把离线收益带回下一次扩建
```

按钮状态：

- 可选路线按钮可点击。
- 已选择后不再显示可点击路线按钮。
- 未解锁星图巡航时不显示该区块。

视觉约束：

- 不使用新大卡片嵌套卡片。
- 不新增顶部导航或新标签页。
- 路线按钮复用共鸣节点按钮的密度和尺寸。
- 文案保持短句，避免把矩阵变成说明书。

## 数值原则

星图巡航第一版只做轻量倾向：

- 路线效果弱于共鸣节点效果。
- 路线效果可以和对应共鸣节点叠加，但不能改变成本曲线。
- 不新增星尘以外的普通资源。
- 不新增可反复购买的路线等级。

效果对比：

| 内容 | 当前效果 | v0.5 路线效果 |
| --- | --- | --- |
| 稳定回路 | 自动采集产出 +10% | 采集航线额外 +5% |
| 回访线圈 | 离线收益 +10% | 回访航线额外 +5% |
| 调校刻印 | 调校倍率 +0.05 | 调校航线额外 +0.03 |

如果 self-playtest 显示 20 小时后等待仍然只是更长空窗，优先补航段目标或回访事件，不直接提高倍率。

## 存档与兼容

v0.5 第一版需要把存档升级到 v3。

新增字段：

```ts
activeCruiseRoute: "collection-route" | "return-route" | "tuning-route" | null
```

兼容策略：

- v1 / v2 存档加载时默认 `activeCruiseRoute = null`。
- 旧存档不应该自动选择路线。
- 如果未来删除或改名路线，未知 route id 应回退为 `null`。
- 离线收益结算要能读取旧存档，不因为缺少路线字段失败。

## 指标边界

第一版可以新增最多 2 个 local-only 指标字段，但不是第一刀必需。

候选指标：

- `cruiseRouteSelectedCount`
- `firstCruiseRouteTime`

不上传，不跨设备，不记录个人数据。若实现第一刀时没有明确运营读回需求，可以先不加指标，避免为了指标扩复杂度。

## 后续内容切片

星图巡航应按小切片扩展，而不是一次做完大系统。

推荐顺序：

1. `v0.5 第一刀`：预算、存档字段、路线选择、轻量效果、目标读回。
2. `v0.5 第二刀`：航段目标读回，例如采集航线目标指向第 28 台自动采集器，回访航线目标指向一次有效离线回访。
3. `v0.5 第三刀`：回访后的巡航事件反馈，例如“回访航线带回一段星尘潮汐”。
4. `v0.5 第四刀`：根据真实反馈或 metrics，决定是否需要路线进度字段。

禁止顺序：

- 第一刀就做完整地图。
- 第一刀就加路线等级。
- 第一刀就做三个航段、三个事件和多个奖励。
- 用第三共鸣门槛替代星图巡航。

## 复杂度预算建议

v0.5 应新增 `20-Hour Content Budget`，但只放开一个很窄的玩法层：

- Primary resource: 1
- Secondary resources: max 1
- Main actions: max 2
- Upgrade types: max 4
- Visible panels: max 5
- First 60 seconds mechanics: max 3
- Resonance resource: allowed
- Resonance matrix panel: max 1
- Star chart cruise: allowed
- Cruise routes: max 3
- Active cruise route: max 1
- Save format versions: max 3
- Local-only cruise metric fields: max 2

v0.5 仍禁止：

- prestige
- 任务系统
- 复杂地图
- 多生产线
- 多个新面板
- 新普通资源
- 第三共鸣门槛
- 新共鸣节点
- 外部 analytics SDK 或上传 telemetry

## 方案比较

### 方案 A：星图巡航

优点：

- 是真正的新决策，不只是第三门槛。
- 复用现有共鸣矩阵和双节点组合。
- 可以逐步扩展到 20 小时内容弧线。
- 不需要立刻引入 prestige 或任务系统。

风险：

- 需要新增存档字段。
- 如果效果过强，会干扰现有经济。
- 如果 UI 表达不克制，会让共鸣矩阵变吵。

### 方案 B：第三共鸣门槛

优点：

- 实现最小。
- 复用现有共鸣逻辑。

风险：

- 本质仍是更远目标，不是新内容。
- 容易把 20 小时拉成重复等待。
- 和用户“内容不足”的反馈不匹配。
- 容易和当前 v0.4 已完成状态冲突，让“当前版本共鸣目标已完成”失去可信度。

### 方案 C：prestige

优点：

- 长线内容感强。
- 能形成新循环。

风险：

- 当前过早。
- 会让玩家以为现有 20 小时进度被清空。
- 需要重做大量经济、UI 和说明。

## 结论

采用方案 A：`星图巡航`。

v0.5 第一版只做最小闭环：

- 新增 v0.5 复杂度预算。
- 新增 `activeCruiseRoute` 存档字段。
- 在第二共鸣和 2/2 节点后显示巡航方向选择。
- 选择一条巡航方向后应用轻量效果。
- 阶段目标读回当前巡航方向和下一步计划。

不做 prestige、任务系统、多生产线、第三共鸣门槛或新共鸣节点。

后续如果继续丰富 20 小时内容，优先扩 `星图巡航` 的航段目标和回访事件，而不是继续堆共鸣门槛。
