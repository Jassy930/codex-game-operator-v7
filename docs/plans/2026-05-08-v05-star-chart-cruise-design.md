# v0.5 星图巡航设计

## 背景

v0.4 把首个共鸣后的长期目标延长到约 20 小时：第二共鸣门槛、第二个现有永久节点、双节点组合读回和当前版本完成读回都已经落地。

但这仍然主要是“更远门槛 + 更清楚文案”。它能解释 20 小时目标，却不足以支撑 20 小时的实际游玩内容。玩家在第二共鸣后仍主要等待自动采集器和调校工具的下一次购买，没有新的决策节奏。

v0.5 的目标是引入一个真正的新内容层，但仍保持轻量 idle / incremental 结构。

## 目标

让第二共鸣后的玩家获得新的长期玩法：

- 不只是看见“当前版本目标已完成”。
- 能基于已启动的双节点组合选择一个巡航方向。
- 巡航方向改变后续 20 小时内的目标、收益倾向和回访计划。
- 第一版只做一个可理解、可测试、可继续扩展的最小闭环。

## 非目标

- 不做 prestige / 重置循环。
- 不做任务系统、每日任务、成就清单或委托列表。
- 不做多地图、多生产线、多角色或复杂探索地图。
- 不新增第三共鸣门槛或新共鸣节点来冒充新内容。
- 不新增外部 analytics、上传 telemetry 或个人数据。
- 不改变前 60 秒核心循环。

## 核心概念

### 星图巡航

`星图巡航` 是第二共鸣后解锁的长期路线选择。

它不是地图系统。第一版只是一条嵌入现有 `共鸣矩阵` 的长期路线状态：

1. 玩家领取第二共鸣。
2. 玩家启动第 2 个永久节点。
3. `共鸣矩阵` 出现 `星图巡航` 选择。
4. 玩家选择一个巡航方向。
5. 该方向改变长期目标文案，并给一个小幅、可解释的长期倾向效果。

### 巡航方向

第一版只根据现有双节点组合提供 2 到 3 个方向，不新增资源。

可用方向：

- `采集航线`：偏向自动采集器，提升长期生产目标感。
- `回访航线`：偏向离线收益和回访节奏。
- `调校航线`：偏向调校工具和倍率成长。

方向不是任务。它是玩家的长期策略姿态。

## 第一版玩法

### 解锁条件

同时满足：

- 已领取首个共鸣。
- 已领取第二共鸣。
- 已启动 2 个现有永久节点。
- 当前没有已选择的巡航方向。

### 选择规则

- 第一版只允许选择 1 条巡航方向。
- 选择后写入存档。
- 选择后可读回，但不能频繁切换。
- 不新增按钮行；复用 `共鸣矩阵` 的紧凑区域。

### 第一版效果

效果必须轻，不破坏经济：

- `采集航线`：自动采集器产出额外 +5%。
- `回访航线`：离线收益额外 +5%。
- `调校航线`：有效调校倍率额外 +0.03。

这些效果比共鸣节点弱，作用是让路线选择被感知，而不是开新系统。

### 目标读回

选择后，`星尘引擎室` 阶段目标应显示：

- 当前巡航方向。
- 它正在强化哪类长期收益。
- 下一次更近升级或回访建议。

示例：

```text
星图巡航：采集航线正在放大自动采集，约 42 分钟后可购买自动采集器
```

## 复杂度预算建议

v0.5 应新增 `20-Hour Content Budget`，但只放开一个很窄的玩法层：

- Primary resource: 1
- Secondary resources: max 1
- Main actions: max 2
- Upgrade types: max 4
- Visible panels: max 5
- First 60 seconds mechanics: max 3
- Resonance resource: allowed
- Resonance matrix panel: max 1
- Star chart cruise: allowed
- Cruise routes: max 3
- Active cruise route: max 1
- Save format versions: max 3
- Local-only cruise metric fields: max 2

v0.5 仍禁止：

- prestige
- 任务系统
- 复杂地图
- 多生产线
- 多个新面板
- 新普通资源
- 第三共鸣门槛
- 新共鸣节点
- 外部 analytics SDK 或上传 telemetry

## 方案比较

### 方案 A：星图巡航

优点：

- 是真正的新决策，不只是第三门槛。
- 复用现有共鸣矩阵和双节点组合。
- 可以逐步扩展到 20 小时内容弧线。
- 不需要立刻引入 prestige 或任务系统。

风险：

- 需要新增存档字段。
- 如果效果过强，会干扰现有经济。
- 如果 UI 表达不克制，会让共鸣矩阵变吵。

### 方案 B：第三共鸣门槛

优点：

- 实现最小。
- 复用现有共鸣逻辑。

风险：

- 本质仍是更远目标，不是新内容。
- 容易把 20 小时拉成重复等待。
- 和用户“内容不足”的反馈不匹配。

### 方案 C：prestige

优点：

- 长线内容感强。
- 能形成新循环。

风险：

- 当前过早。
- 会让玩家以为现有 20 小时进度被清空。
- 需要重做大量经济、UI 和说明。

## 结论

采用方案 A：`星图巡航`。

v0.5 第一版只做最小闭环：

- 新增 v0.5 复杂度预算。
- 新增 `activeCruiseRoute` 存档字段。
- 在第二共鸣和 2/2 节点后显示巡航方向选择。
- 选择一条巡航方向后应用轻量效果。
- 阶段目标读回当前巡航方向和下一步计划。

不做 prestige、任务系统、多生产线、第三共鸣门槛或新共鸣节点。
