# Soft Automation Design

## 背景

当前游戏已经有单一资源星尘、手动采集、自动采集器、购买进度、动态目标、里程碑、离线收益和反馈入口。Issue #1 指向的采集动机问题已经通过动机文案和采集反馈做了最小修复。

下一步需要一个低复杂度玩法候选，让玩家在不离开主循环的情况下获得更多选择感。

## 方案比较

推荐方案：`调校工具`。玩家用星尘购买第二种升级，提升每台自动采集器的效率。它保留单一资源和现有主屏，让玩家在“买更多自动采集器”和“强化已有自动采集器”之间选择。

备选方案：增加点击收益升级。实现更简单，但会把玩家继续推向高频点击，不如强化 idle / automation 方向。

备选方案：增加第二资源或 prestige。它能提供长期目标，但当前阶段复杂度过高，会违反 North Star 中“更深入但不更吵”的约束。

## 设计边界

- 只新增一个 upgrade type：`调校工具`。
- 不新增资源、面板、反馈渠道、analytics 或个人数据。
- `调校工具` 只影响自动采集器效率，不影响点击收益。
- 第一版只需要可见成本、购买按钮、当前效率倍率和测试覆盖。
- 实现前必须确认存档兼容策略，避免破坏 version 1 save。

## 数据模型候选

- `autoCollectorEfficiencyLevel`: 从 0 开始。
- `autoCollectorEfficiencyMultiplier`: 初始 1，每级增加一个小倍率。
- `nextEfficiencyUpgradeCost`: 使用星尘成本，增长速度应慢于或接近自动采集器成本，避免吞掉第一分钟目标。

## UI 候选

在现有动作区中增加一个次级升级按钮，文案类似：

`调校工具 · 自动采集效率 +10% · 需要 25 星尘`

已有统计区可以显示当前效率倍率；若这会增加首屏噪音，则先只在按钮文案中表达，不新增统计格。

## 测试要求

- 游戏逻辑测试：购买调校工具会花费星尘并提高自动采集器产出。
- 不足星尘时不能购买。
- 存档加载测试：旧存档缺少新字段时应使用默认值。
- UI 测试：按钮文案包含成本和效率提升。

## 不在本设计中做

- 第二资源。
- prestige。
- 多 generator 分类。
- 成就、任务、技能树。
- 新面板或弹窗教程。
