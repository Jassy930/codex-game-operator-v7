# v0.7 归航航线当前收益读回设计

## 背景

`归航航线` 已能显示当前段、下一段门槛、动态差距、下一步行动和下一段预览。复核后仍有一个轻量可读性缺口：玩家能知道下一段去哪里，但需要从余辉读回和描述文案里推断当前段已经带来什么长期收益。

## 设计

在 `getReturnRouteReadback` 中新增纯派生字段 `currentPayoff`，按现有三段航线返回当前收益文案：

- `余辉起航`：额外共鸣已能转成下一轮起步星尘。
- `稳航校准`：余辉开局已稳定，重复归航会继续压缩重建时间。
- `深空归航`：三段航线已贯通，额外共鸣会作为后续版本储备。

UI 继续复用现有 `共鸣矩阵` 的 `归航航线` 块，在段落说明后显示当前收益。阶段目标保持优先读回行动提示，避免目标行过长。

## 约束

不新增第三普通资源、第三共鸣门槛、新共鸣节点、节点等级、节点等级树、任务系统、多生产线、额外面板、存档字段、指标字段、视觉资产或 telemetry。不修改 `星尘归航` 奖励、`归航余辉` 公式或 50 星尘上限。

## 验证

- `src/return-route.test.ts` 覆盖三段航线的 `currentPayoff`。
- `src/App.test.tsx` 覆盖 `共鸣矩阵` 中的当前收益读回。
- 标准收口链路仍为 `bun test`、`bun run test`、`bun run build`、`./ops/governor-check.sh`、`git diff --check`。
