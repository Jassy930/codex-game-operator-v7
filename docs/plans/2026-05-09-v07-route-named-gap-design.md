# v0.7 归航航线命名差距设计

## 背景

`归航航线` 已经能在现有 `共鸣矩阵` 中显示当前段、量化收益、距下一段差距、行动提示、节奏预判、航线图和下一段预览。复核发现：摘要和阶段目标里的“下一段”仍偏抽象，玩家需要再读下一段预览才知道自己正在冲 `稳航校准` 还是 `深空归航`。

这个缺口不会靠新增系统解决。更小的修法是把目标段名称直接压进现有差距读回。

## 本轮选择

继续保持纯派生读回，只调整 `progressSummary` 和 `cadenceForecast` 的文案：

- 第一段未完成时显示“距稳航校准还差 ...”。
- 第二段未完成时显示“距深空归航还差 ...”。
- 节奏预判同步显示“进入稳航校准 / 深空归航”。
- `routeSummary`、`共鸣矩阵` 详情和阶段目标继续复用同一个派生读回。

## 明确不做

- 不新增资源、节点、节点等级、任务系统、多生产线、额外面板、存档字段、视觉资产或 telemetry。
- 不修改 `星尘归航` 奖励。
- 不修改 `归航余辉` 公式或 50 星尘上限。
- 不改变 v0.7 最多 3 段航线的复杂度边界。

## 验证

- `src/return-route.test.ts` 覆盖第一段、第二段和只缺单一条件时的命名差距。
- `src/App.test.tsx` 覆盖矩阵 SSR 读回和阶段目标。
- 完整运行 `bun test`、`bun run test`、`bun run build`、`./ops/governor-check.sh`、`git diff --check`。
