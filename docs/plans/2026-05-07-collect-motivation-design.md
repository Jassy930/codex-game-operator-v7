# 采集动机反馈设计

## 背景

Issue #1 是第一条真实玩家反馈。玩家的问题集中在前 60 秒：不知道为什么要采集星尘、采集会带来什么，以及采集本身的兴奋点不足。

## 方案比较

推荐方案：在现有首屏内增加一条短动机文案，并在点击采集后显示短暂反馈。它直接回答“星尘会变成自动采集器，让工坊持续产出”，不改变经济、不新增系统，符合当前复杂度预算。

备选方案：只回复 issue 询问更多信息。该反馈已经足够具体，继续追问会延迟可验证改进。

备选方案：新增奖励、技能树或额外资源。该方向可能提升兴奋点，但超出当前 early clean-room 阶段，也会违反“先强化前 60 秒清晰度”的约束。

## 设计

首屏在星尘读数下方展示一条动机文案：星尘会变成自动采集器，让工坊持续产出。该文案不新建面板，保持在现有 resource readout 内。

点击“采集”后，复用已有 `event-stack` 显示短暂反馈，说明本次星尘已进入下一台自动采集器进度。该反馈和购买确认一样会自动消失，不阻断操作，不改变数值。
