# Content Arc

本文档定义 `星尘工坊` 的低复杂度内容弧线。它不是任务清单；只有通过 `docs/DECISION.md` 授权后，才可以转化为实现。

## v0.2 目标

让玩家在 3-15 分钟内感觉工坊正在变深，但界面不变吵。

约束：

- 只保留 1 个主资源：星尘。
- 不新增第二资源、prestige、任务系统或复杂地图。
- 优先复用现有主屏、现有升级和现有反馈区域。
- 新内容必须说明使用了 `docs/COMPLEXITY_BUDGET.md` 的哪一项预算。

## v0.3 目标

让首次回访后的玩家获得一个真实策略层，而不是只看到更远的里程碑。

约束：

- 星尘仍是主资源。
- 新增第二资源 `共鸣`，但只从长期生产节点获得。
- 新增 1 个紧凑 `共鸣矩阵` 面板。
- v0.3 第一版只允许三选一永久节点，不引入 prestige 或任务系统。
- 新内容必须保持前 60 秒体验不变。

## 时间窗

| 时间窗 | 玩家应感受到 | 当前表达 | 后续允许的小切片 |
| --- | --- | --- | --- |
| 0-60 秒 | 我知道为什么采集，以及第一台自动采集器快到了 | 星尘动机文案、采集反馈、第一目标、购买进度 | 只做回归修复，不再叠加说明 |
| 1-3 分钟 | 我可以继续扩建，也可以开始调校效率 | 目标提示、调校工具、调校倍率 | 更清楚地显示最近升级目标 |
| 3-15 分钟 | 工坊进入阶段推进，而不是只有数字上涨 | 工坊阶段、下一阶段条件、物件插图 | 阶段完成反馈、延后解锁文案 |
| 15-60 分钟 | 我知道首次长期生产线是否形成，并愿意做首次回访 | `星尘引擎室` 长期目标 | 先用 self-playtest 或 metrics 证明 gap |
| 首次回访 | 离线收益能被理解为工坊持续生产 | 离线收益提示、本地 session history | 回访文案或本地 metrics readback |
| v0.3 回访后 | 我可以把长期生产转化为新的永久选择 | 已实现：共鸣资源、共鸣矩阵、首个共鸣里程碑和三选一节点 | 继续只允许小切片，不直接追加 prestige 或任务系统 |

`DECISION:2026-05-07-v03-resonance-system`

## 上一阶段复核

`DECISION:2026-05-07-research-milestone-resonance`

下一阶段内容研究结论：`星尘引擎室` 后不直接进入第二资源、prestige、主动 boost 或新面板。下一步候选是 `Milestone Resonance / 星尘共鸣节点`：用现有自动采集器数量和调校等级派生新的长期里程碑，先作为预告或派生状态，让玩家在首次回访后知道下一组结构化目标。

这不是实现授权。实现前必须再次确认：

- 是否只做预告，还是影响自动产出倍率。
- 是否仍不需要 save version 2。
- 是否能复用现有阶段/目标/事件反馈区域。
- 是否有 self-playtest gap、真实反馈或 metrics readback 支撑。

## 上一阶段复核

`DECISION:2026-05-07-offline-return-after-spend-goal`

首次回访后消费模拟显示，玩家可以把离线收益连续投入第 16 台自动采集器和第 10 次调校。消费后离线收益提示仍可见，但当前星尘已低于下一次升级成本。因此 `星尘引擎室` 在离线收益实际可见时，会先提示花掉离线星尘；当当前没有可负担升级时，同一阶段目标行切换为“离线收益已投入工坊，继续攒下一次升级”。本轮不新增第三或第四种升级，也不新增资源、按钮、面板、存档字段、指标字段或反馈渠道。

下一步如果继续推进，必须先满足至少一个条件：

- 玩家在 Issue #1/#2 或新 issue 中补充新的具体卡点。
- 真实 localStorage session history 暴露回访或升级节奏异常。
- 首次回访后 self-playtest 证明消费离线收益后的下一步仍不清楚。
- research-backed 决策证明一个更小切片能改善内容弧线，并通过复杂度预算。

## 当前小切片

`DECISION:2026-05-07-offline-return-after-spend-goal`

`星尘引擎室` 的回访目标现在会区分两种状态：

- 离线收益可见且当前仍能购买升级：提示花掉离线星尘，继续扩建或调校。
- 离线收益可见但当前没有可买升级：提示离线收益已投入工坊，继续攒下一次升级。

## 进入下一轮前

v0.3 共鸣系统第一版已经实现。下一轮若继续扩展，必须先复核共鸣门槛、节点选择和本地指标，不得直接引入 prestige、任务系统、多生产线或第二个新面板。必须先满足至少一个条件：

- 新玩家反馈指出 3-15 分钟仍缺少目标。
- self-playtest 证明阶段条件或最近升级目标仍无法解释玩家下一步。
- local-only metrics 显示长期回访或升级节奏异常。
- research-backed 决策证明一个更小切片能改善内容弧线。
