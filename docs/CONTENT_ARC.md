# Content Arc

历史内容弧线细节已归档到 `docs/archive/2026-05-08-runtime-docs/CONTENT_ARC.md`。本文件记录当前可执行方向。

## North Star Fit

`星尘工坊` 应让玩家持续感到：

- 我知道下一步做什么。
- 下一次升级在可理解范围内。
- 离开后回来仍能读懂进度。
- 游戏在变深，但不变吵。

## Current Arc

### 0-60 秒

目标：清楚理解采集、星尘、第一台自动采集器和自动保存。

护栏：共鸣矩阵、归航和长期机制不得干扰首分钟。

### 3-15 分钟

目标：通过自动采集器、调校工具、工坊阶段和下一升级进度形成轻量内容弧线。

已完成：阶段名、下一阶段条件、当前进度、购买跨阶段反馈和物件插图。

### 首次回访

目标：玩家能理解离线星尘、花掉回访收益和下一次等待目标。

已完成：离线收益提示、回访目标、等待时间读回和 local-only metrics snapshot。

### 4-20 小时

目标：用 `共鸣矩阵`、`星尘归航` 和轻量 `归航余辉` 形成低复杂度长线循环。

已完成：首个共鸣、最多 2 个永久节点、20 小时巡航读回、归航按钮、归航奖励 `1 共鸣`、归航后目标提示、第 2 轮开局共鸣矩阵读回、满节点后额外共鸣暂存说明，以及 v0.6 `归航余辉` 的 capped 开局星尘助推。

当前读回：余辉会把起步星尘换算为可立即重建的自动采集器数量；50 星尘上限对应前 3 台自动采集器，让重复归航后的新一轮更快回到 `星尘小间`。满节点且已有归航历史后，阶段目标会优先显示当前 `归航航线`。

### 20 小时后

目标：用 v0.7 `归航航线` 解释重复归航正在推进的长期方向，而不是让玩家只看到余辉变多。

已完成第一段可玩读回、本段进度、贯通态储备数量、具名动态差距读回、行动提示、下一段预览、当前收益读回、余辉已投入读回、具名节奏预判、航线图和航线摘要：最多 3 个航线里程碑，复用现有 `共鸣矩阵` 和阶段目标读回；航线从 `returnCount`、额外共鸣和已启动永久节点派生。未贯通时会先用摘要读回当前段、量化收益、距 `稳航校准` / `深空归航` 还差什么，以及按当前节奏会进入哪一段，收益会换算成“下轮起步可重建 N 台自动采集器”，再显示当前归航次数 / 目标、额外共鸣 / 目标、下一步行动、下一段预览、完整三段路径和当前收益；三段贯通后会显示累计归航次数和额外共鸣储备；当余辉起步星尘已被花掉时，矩阵仍说明余辉已投入重建节奏。不新增资源、节点、节点等级树、任务系统、额外面板、存档字段或 telemetry。
