# Research Policy

## Purpose

Web research is an input source, not a feature generator.

## When Research Is Allowed

Use RESEARCH mode when:

- no real feedback exists;
- feedback is ambiguous;
- a product question is unclear;
- a mechanic needs design references;
- self-playtest identifies a problem but not a solution.

## Scope

Research must answer one question only.

Examples:

- How fast should the first upgrade arrive in an idle game?
- How do clicker games show next goals?
- How do incremental games avoid early confusion?
- How should offline rewards be communicated?

## Required Output

Research must update `RESEARCH.md` with:

1. Research question
2. Sources / observations
3. General principles
4. Decision impact
5. What not to copy

## Rule

Research cannot directly create implementation tasks.

Only `DECISION.md` can authorize implementation.
