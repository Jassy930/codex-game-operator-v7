# 《星尘工坊》游戏界面重设计实现说明

> 用途：提供给 Codex / Claude Code / Cursor 作为 UI 重设计实现依据。  
> 目标：参考两张设计图，重构当前游戏界面，使其从“网页卡片感”升级为“真正可落地的 Idle / Incremental Game 控制台界面”。  
> 重点：**只做界面重设计、信息架构重排、交互反馈增强，不改变当前游戏机制、存档结构和核心状态逻辑。**

---

## 1. 参考设计图

本次界面重设计需要同时参考两张图：

1. **桌面端参考图**
   - 文件建议命名：`desktop-ui-reference.png`
   - 目标：宽屏三栏布局，适合 1440px 以上桌面端。
   - 关键词：星尘工坊、暖科幻、控制台、横向 HUD、左侧导航、中间主操作、右侧目标与状态。

2. **移动端参考图**
   - 文件建议命名：`mobile-ui-reference.png`
   - 目标：移动端单列纵向布局，适合 375px - 430px 屏幕。
   - 关键词：首屏可点击、资源数字突出、采集按钮明显、底部导航、卡片堆叠。

实现时不要像素级复刻图片，而是提取它们的 **信息架构、视觉层级、交互优先级、色彩方向和游戏 UI 感**。

## 2. 当前项目实现边界

当前项目是一个轻量 Web Idle / Incremental Game。界面重设计必须服从当前项目约束。

### 允许做

- 重构 `src/App.tsx` 的 JSX 结构。
- 重写 / 扩展 `src/styles.css`。
- 新增纯展示组件。
- 新增小型格式化 helper。
- 调整文案。
- 调整布局、样式、响应式。
- 增加 CSS 动效，例如 hover、active、发光、轻微浮动。
- 增加非持久化、非复杂化的临时事件展示。

### 不允许做

- 不改变 `GameState` 类型。
- 不改变存档 key。
- 不改变 serialize / hydrate / migration 逻辑。
- 不改变核心游戏函数语义。
- 不新增资源类型。
- 不新增共鸣节点。
- 不新增任务系统。
- 不新增多生产线。
- 不新增完整科技树。
- 不新增完整日志持久化。
- 不新增外部 telemetry。
- 不新增 UI 框架。
- 不引入 Tailwind、shadcn、framer-motion 或其他新依赖。

### 必须保持通过

```bash
bun run test
bun run build
```

## 3. 设计目标

把当前界面从“网页卡片 / MVP 管理面板”改成真正的 **星尘工坊游戏控制台**。

玩家打开游戏后，应该在 5 秒内理解：

1. 当前有多少星尘。
2. 每秒产出是多少。
3. 点击哪里采集星尘。
4. 下一步应该购买什么。
5. 自动采集器和调校系统如何成长。
6. 共鸣矩阵和星尘归航是中后期系统。
7. 游戏未来会扩展研究所、日志、统计等模块，但当前版本不实装这些系统。

---

## 4. 核心体验循环

界面必须围绕这个主循环组织：

```text
采集星尘
  → 购买自动采集器
  → 提升调校效率
  → 达成里程碑
  → 领取共鸣
  → 解锁共鸣节点
  → 启动星尘归航
  → 开始更强的一轮
```

首屏必须服务于前 3 步：**星尘数量、采集按钮、下一项工程、升级卡片**。

共鸣、归航、研究所、日志等入口可以可见，但不能抢占主循环。

## 5. 总体视觉方向

### 关键词

- 暖科幻
- 星尘工坊
- 游戏控制台
- 深蓝夜空
- 暖金星尘
- 青蓝科技光
- 紫色共鸣能量
- 卡片式 HUD
- 明确按钮反馈
- 可落地，不做纯海报

### 色彩方向

建议在 CSS 中定义变量：

```css
:root {
  --bg-night: #06111f;
  --bg-panel: rgba(9, 24, 42, 0.82);
  --bg-panel-strong: rgba(13, 31, 55, 0.94);
  --border-soft: rgba(255, 205, 112, 0.24);

  --text-main: #fff1c7;
  --text-muted: #aebbd0;

  --gold: #ffc85a;
  --gold-soft: rgba(255, 200, 90, 0.18);

  --cyan: #3fd6ff;
  --cyan-soft: rgba(63, 214, 255, 0.16);

  --violet: #a875ff;
  --violet-soft: rgba(168, 117, 255, 0.16);

  --teal: #31e6d0;
  --teal-soft: rgba(49, 230, 208, 0.16);

  --success: #7ef29a;
  --disabled: rgba(160, 170, 190, 0.36);
}
```

### 风格要求

- 大数字有游戏资源感。
- 卡片有深色半透明面板、细边框、轻微内发光。
- 主要按钮有明显可点击感。
- 主资源区可以有背景图，但不能像宣传海报。
- 图片素材要被用作 UI 图标 / 装饰，而不是撑满整个画面。
- 所有交互元素都要像游戏按钮，而不是普通网页链接。

## 8. 界面区域详细说明

### A. 顶部 HUD

#### 作用

提供全局身份、自动保存状态、阶段、长期资源和功能入口。

#### 推荐文案

```text
星尘工坊
自动保存中
阶段 3：星尘汇聚
共鸣 275
第 3 次归航
```

#### 功能按钮

```text
统计
反馈
设置
菜单
```

#### 当前实现建议

- “反馈”继续使用现有反馈逻辑。
- 设置、菜单、统计可以先做视觉按钮，不实现复杂功能。
- 不要新增设置面板。

#### 视觉要求

- 高度紧凑。
- 横向排布。
- 资源用胶囊按钮样式。
- 自动保存用绿色小圆点。

---

### B. 左侧导航栏 / 移动端底部导航

#### 作用

表达系统分区和未来扩展方向。

#### 当前可用入口

```text
引擎室
共鸣室
归航台
```

#### 当前锁定入口

```text
研究所
日志
```

#### 交互说明

第一版不需要做真实路由。可以做锚点滚动：

- 点击“引擎室”：滚动到星尘核心。
- 点击“共鸣室”：滚动到共鸣矩阵。
- 点击“归航台”：滚动到星尘归航。
- 点击“研究所 / 日志”：显示 locked 状态，不做功能。

#### 设计意图

给玩家“这是一个有系统成长空间的游戏”的感觉，但不实际增加复杂玩法。

---

### C. 星尘核心区

#### 作用

当前最重要资源展示区。

#### 推荐文案

```text
星尘核心
12,840.5
星尘

每秒产出 +18.2
每次点击 +1
倍率 x1.00
```

#### 对应状态

- `state.dust`
- `state.dustPerSecond`
- `state.dustPerClick`
- `getEffectiveAutoCollectorEfficiencyMultiplier(state)`

#### 视觉说明

- 大数字是最高视觉优先级。
- 右侧或背景展示星尘水晶。
- 水晶可以发光，但不能压过数字。
- 资源卡片必须像 UI 面板，不像单张插画。

---

### D. 主采集按钮

#### 作用

第一主交互按钮。

#### 推荐文案

```text
采集星尘 +1
```

#### 对应行为

继续调用现有点击采集函数。

#### 交互说明

- hover：按钮略微变亮。
- active：按钮轻微下压。
- 点击后可以显示临时反馈：
  - `+1 星尘`
  - `采集了 +1 星尘`

#### 视觉要求

- 页面最大按钮。
- 暖金色。
- 必须明显可点击。
- 不能做成单纯装饰条。

---

### E. 自动采集器升级卡

#### 作用

第一个成长目标，引导玩家进入自动产出。

#### 推荐文案

```text
建造自动采集器
自动采集星尘，持续为工坊提供资源。

当前拥有：12 台
每秒产出：+18.2

150 星尘
```

#### 对应状态

- `state.autoCollectors`
- `state.nextAutoCollectorCost`
- `state.dustPerSecond`
- `canBuyAutoCollector`

#### 对应行为

继续调用当前购买自动采集器逻辑。

#### 不足时文案

```text
还差 32 星尘
```

#### 视觉方向

- 青蓝色卡片。
- 使用自动采集器图标。
- 可购买时按钮高亮。
- 不可购买时按钮 disabled，但价格仍清楚。

### F. 调校工坊频率升级卡

#### 作用

放大自动产出效率。

#### 推荐文案

```text
调校工坊频率
提升工坊整体效率，放大所有产出。

当前倍率：1.8x
下一等级：2.0x

250 星尘
```

#### 对应状态

- `state.autoCollectorEfficiencyLevel`
- `state.autoCollectorEfficiencyMultiplier`
- `state.nextEfficiencyUpgradeCost`
- `getEffectiveAutoCollectorEfficiencyMultiplier(state)`
- `canBuyEfficiencyUpgrade`

#### 对应行为

继续调用当前调校升级逻辑。

#### 无自动采集器时文案

```text
需要先建造自动采集器
```

#### 视觉方向

- 紫色 / 蓝紫色卡片。
- 使用调校工具图标。
- 强调倍率变化。

#### 作用

### J. 事件记录

#### 作用

给玩家操作反馈，避免“点击后没感知”。

#### 推荐文案

```text
事件记录

离线收益 +2,450 星尘
购买了自动采集器
升级了调校工坊频率
采集了 +1 星尘
```

#### 当前落地方式

不要新增持久日志数组。只展示已有临时事件：

- `offlineDust`
- `collectMessage`
- `purchaseMessage`

无事件时显示：

```text
工坊运行稳定，等待下一次操作
```

#### 重要约束

不要写入 localStorage。  
不要实现完整日志系统。  
不要新增 telemetry。

### K. 共鸣矩阵

#### 作用

中后期长期成长系统。

#### 推荐文案

```text
共鸣矩阵
可用共鸣：1

稳定回路
Lv.2
自动采集产出 +10%

回访线圈
可领取
离线收益 +10%

调校刻印
未解锁
调校倍率 +0.05
```

#### 交互说明

- 已解锁节点显示 active。
- 可领取共鸣显示“领取奖励”。
- 可升级节点显示“升级”。
- 不可解锁节点显示锁定条件。
- 点击节点按钮继续调用当前共鸣相关逻辑。

#### 视觉方向

- 比普通升级卡更高级。
- 使用紫色、青色、金色混合。
- 已激活节点有发光边框。
- 未解锁节点灰化并显示锁图标。

#### 重要约束

- 不新增共鸣节点。
- 不突破当前最大可解锁节点数量。
- 不实装技能树。

### L. 星尘归航

#### 作用

表达 prestige / reset 循环。

#### 推荐文案

```text
启动星尘归航
重启本轮工坊，保留共鸣和永久节点。

本次归航可获得：+1 共鸣
```

如果有新一轮起步星尘：

```text
共鸣余辉：新一轮从 50 星尘起步
```

#### 对应状态

- `canReturn`
- `state.returnCount`
- `returnAfterglowDust`
- `hasActiveReturnAfterglow`
- `hasParkedReturnResonance`

#### 交互说明

- 可归航时：按钮高亮。
- 不可归航时：展示条件进度。
- 点击后继续调用当前星尘归航函数。

#### 重要约束

当前不应该新增“归航余辉”资源字段。  
如果界面中出现“归航余辉”，应作为文案概念处理，而不是新状态。

- 左侧导航固定宽度。
- 中央主操作区最大化。
- 右侧状态区固定最小宽度。

### 平板端

- `900px - 1100px`：可以两栏或单栏。

## 14. 验收标准

- 点击“采集星尘”仍然增加星尘。

- 星尘归航逻辑不变。
- 自动保存逻辑不变。
- 离线收益逻辑不变。
- 存档格式不变。

### UI 验收

- 桌面端像游戏控制台，不像网页表单。
- 移动端像游戏界面，不像压缩版海报。
- 星尘数量是最强视觉层级。
- 采集按钮是最强交互层级。
- 下一项工程始终清楚。
- 升级卡一眼能看懂价格和状态。
- 共鸣矩阵像中后期系统，但不复杂。
- 星尘归航像长期循环目标。
- 研究所 / 日志体现未来空间，但保持锁定。

### 响应式验收

- 1440px 宽度下布局稳定。
- 1024px 宽度下无明显溢出。
- 375px 宽度下无横向滚动。
- 移动端首屏可见主资源和采集按钮。
- 底部导航不遮挡核心按钮。

### 工程验收

- `bun run test` 通过。
- `bun run build` 通过。
- 不新增依赖。
- 不新增复杂状态。
- 不破坏现有测试。
