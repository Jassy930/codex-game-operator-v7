# Response Budget

## 目的

防止 issue 讨论串变成产品方向的驱动器。

## 全局预算

每个自治周期：

- 最多回复 issue：1 次
- 最多新建 issue：1 个
- 默认：0 次回复

## 单个 Issue 预算

每个 issue 通常最多获得：

1. 一次确认收到，如果确实有帮助。
2. 一次发布或更新回复，如果确实有变更发布。

除非满足以下情况，否则不继续回复：

- 玩家补充了实质性新信息；
- issue 是 critical；
- 发生了回归；
- 维护者明确改变政策。

## 回复要求

在回复“已修复”“已改进”或类似内容之前，必须全部满足：

- issue 已记录在 `ISSUE_LEDGER.md`；
- issue 已归入聚类，或被标记为 critical；
- `DECISION.md` 引用了该 issue 或聚类；
- 代码或文档已变更；
- commit 已存在，或即将创建；
- `RELEASE_LOG.md` 已更新；
- 部署状态明确。

## 沉默是有效策略

operator 可以保持沉默。

以下情况优先保持沉默：

- issue 含糊；
- issue 偏离策略；
- issue 重复已有聚类；
- 没有真实变更发布；
- 回复会延长低价值讨论。

## 禁止

- 重复同一确认回复；
- 只是为了显示活跃而回复；
- 在没有 roadmap 支撑时承诺未来工作；
- 为了满足单个 issue 作者而反复修改游戏。
